Timeline Interaction

Timelines present events in chronological order. They are common in history, process, and onboarding modules. A good timeline should allow learners to navigate easily between events and understand the sequence of time.

Practice Task

1. Navigate to the timeline track and identify the functional cues that describe its role and orientation.

2. Use the Down and Up arrow keys and identify how the screen reader announces the transition between dates and the sequence (X of Y).

3. Navigate to a specific date and Tab into its details, identifying the cues that confirm you have moved into the related content area.

Inaccessible Example

This timeline is built using a series of floating div elements. It relies entirely on visual position to convey order and has no semantic structure.

Why it's broken

  • No List Semantics: NVDA sees this as a stream of text. It doesn't announce "List with 4 items," so the user doesn't know the scope of the timeline.
  • Poor Navigation: A keyboard user has to read through every word to find the next date. There is no way to quickly "jump" between events.
  • No Relationship: The dates and the descriptions are just text blocks. There is no structural link between the "point in time" and the "event."

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The timeline elements lack appropriate roles and states to describe their interactive nature.
  • SC 1.3.1: Info and Relationships - The chronological sequence and grouping of events are not programmatically defined.

Accessible Example

The accessible version uses a Vertical Tablist. The events are the "tabs," and the descriptions are the "panels." This allows for rapid navigation via arrow keys.

1999: WCAG 1.0 Adoption

WCAG 1.0 was published by the W3C, establishing the first comprehensive set of guidelines for web accessibility, with a strong focus on making HTML content perceivable.

2008: WCAG 2.0 and POUR

WCAG 2.0 introduced the four core principles of accessibility: Perceivable, Operable, Understandable, and Robust (POUR), creating a technology-neutral framework.

2018: WCAG 2.1 Expansion

WCAG 2.1 expanded the guidelines to include critical criteria for mobile accessibility, low vision, and cognitive disabilities.

2023: WCAG 2.2 Standards

WCAG 2.2 continues the evolution of the standards, adding nine new success criteria to improve accessibility for users with cognitive or learning disabilities, and on mobile devices.

How it's fixed

  • Vertical Tablist Pattern: Using role="tablist" and aria-orientation="vertical" tells NVDA exactly how to interact with the timeline.
  • Arrow Key Traversal: Learners can use the Up and Down arrow keys to quickly move through time. This is much faster than tabbing through every element.
  • Explicit Grouping: Each event's details are in a tabpanel, creating a clear programmatic relationship between the date and the content.

Compliance Success

  • SC 4.1.2: Name, Role, Value - Implementing the tablist pattern provides clear roles and selection states.
  • SC 1.3.1: Info and Relationships - Explicit grouping via tabpanels ensures the relationship between the date and its content is preserved.
