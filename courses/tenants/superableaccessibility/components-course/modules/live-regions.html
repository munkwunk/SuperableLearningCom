Live Region Notifications

Live regions allow you to announce dynamic changes to the page that are not caused by a user's direct navigation. These are essential for success messages, error alerts, and status updates. The key is knowing when to be "polite" and when to be "assertive."

Practice Task

1. Initiate a screen reader announcement (e.g., by reading a paragraph).

2. While the screen reader is speaking, press the "Polite" button and identify how it queues the announcement.

3. Repeat the process with the "Assertive" button and identify the functional difference in how the interruption is handled.

Inaccessible Example

In this broken version, clicking the buttons adds a message to the screen visually, but because there are no aria-live attributes, NVDA remains completely silent.

Why it's broken

  • Silent Feedback: The dynamic text is added to the DOM, but NVDA does not monitor standard div or p tags for changes. The user has no idea if their action was successful.
  • Missing Role: Without a live region role (like status or alert) or the aria-live attribute, dynamic content is "invisible" to screen readers until the user manually navigates to it.

Compliance Failures

  • SC 4.1.3: Status Messages - Dynamic updates to the page are not announced to screen reader users, providing no feedback on the results of their actions.

Accessible Example

The accessible version uses two distinct live regions. One is Polite for standard updates, and the other is Assertive for critical interruptions.

How it's fixed

  • Aria-live="polite": Used for the "Success" message. NVDA will wait until it finishes its current sentence before announcing the update. This is the preferred setting for 90% of notifications.
  • Aria-live="assertive": Used for the "Error" message. NVDA will immediately interrupt whatever it is saying to announce the critical information. Use this sparingly!
  • Visual/Auditory Sync: The notification appears both visually (as a toast) and is announced to the screen reader at the same time.

Compliance Success

  • SC 4.1.3: Status Messages - aria-live attributes (polite and assertive) ensure that status updates and error messages are announced to assistive technology.
