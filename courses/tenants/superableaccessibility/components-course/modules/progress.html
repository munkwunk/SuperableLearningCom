Progress Bar / Step Tracker

Progress bars give users a visual indication of how much of a task or course they have completed. To be accessible, they must programmatically communicate their status and current value to screen readers.

Practice Task

1. Navigate to the accessible progress bar and identify its label, role, and current value.

2. Activate the "Increase Progress" button and identify the functional cues that announce the value change.

3. Observe how the component provides both visual feedback and programmatic state updates simultaneously.

Inaccessible Example

This progress bar is just a div with a background color. It is completely invisible to screen readers, and its value is only conveyed through color and width. (Visual only: The bar below is approximately 65% full.)

Why it's broken

  • Silent Updates: Clicking the button updates the bar visually, but NVDA remains silent. The user has no idea if the action was successful or what the new value is.
  • No Role: NVDA sees this as an empty div. It doesn't know it's a progress indicator.
  • No Value: There are no aria-valuenow or aria-valuemax attributes to tell the user the current percentage.
  • No Label: The progress bar isn't linked to a label, so the user doesn't know what progress is being tracked.

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The progress indicator is implemented as a generic div, providing no programmatic role, name, or value.
  • SC 4.1.3: Status Messages - Visual state changes (progress updates) are not announced to screen reader users.

Accessible Example

The accessible version uses role="progressbar". Notice how NVDA announces the percentage and the label as soon as you navigate to it.

How it's fixed

  • Role="progressbar": Identifies the element's purpose to the accessibility tree.
  • Aria-valuenow: Programmatically provides the current value.
  • Aria-labelledby: Connects the progress bar to its descriptive label.

Compliance Success

  • SC 4.1.2: Name, Role, Value - Use of role="progressbar" and ARIA attributes ensures purpose and state are communicated.
