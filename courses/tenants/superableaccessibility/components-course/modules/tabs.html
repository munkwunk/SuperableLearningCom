Tabs

Tabs are used to organize content into related groups that can be viewed one at a time. This is a complex pattern because it requires specific ARIA roles and keyboard interactions (arrow keys) to be fully accessible.

Practice Task

1. Navigate to the accessible tab list and identify the functional cues that describe its role and the currently selected tab.

2. Use the Right and Left arrow keys and identify how the screen reader announces the state change and position (X of Y).

3. Tab into the tab content and identify the functional cues that confirm focus has moved to the related panel.

Inaccessible Example

This example uses simple links. While keyboard users can reach them, NVDA does not identify them as tabs, and there is no way to know which tab is currently "active."

Perceivable Operable Understandable Robust

Perceivable: Presentable to users in ways they can perceive.

Why it's broken

  • Incorrect Roles: NVDA announces these as "link," not "tab." Users don't expect the arrow keys to work.
  • No State: There is no aria-selected attribute, so the screen reader cannot tell the user which tab is active.
  • Silent Updates: The content updates visually, but NVDA users must manually navigate to the content area to discover the change. To the learner, clicking the link appears to do nothing at first.
  • Tab Order Issues: Users have to Tab through every single link to get to the content, rather than using arrow keys to skip between tabs.

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The tabs use link roles instead of tab roles and lack selection state.
  • SC 2.1.1: Keyboard - The tab interface does not support standard arrow key navigation.
  • SC 1.3.1: Info and Relationships - The relationship between the tab and its panel is not programmatically defined.

Accessible Example

The accessible version implements the ARIA Tabs pattern. It uses role="tablist", role="tab", and role="tabpanel".

Perceivable

Information and user interface components must be presentable to users in ways they can perceive.

Operable

User interface components and navigation must be operable.

Understandable

Information and the operation of the user interface must be understandable.

Robust

Content must be robust enough that it can be interpreted by a wide variety of user agents, including assistive technologies.

How it's fixed

  • Standard Roles: Using tablist and tab roles tells NVDA exactly how to behave.
  • Arrow Key Navigation: Only the active tab is in the Tab order (tabindex="0"). Other tabs are reached via arrow keys, reducing keyboard clutter.
  • Dynamic Labels: aria-selected="true" provides immediate feedback when a tab is activated.

Compliance Success

  • SC 4.1.2: Name, Role, Value - ARIA tab roles and states (aria-selected) are correctly applied.
  • SC 2.1.1: Keyboard - Standard arrow key navigation is implemented.
  • SC 1.3.1: Info and Relationships - Tabs are programmatically linked to their respective panels.
