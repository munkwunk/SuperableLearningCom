Tooltip / Info Hover

Tooltips provide extra context for icons or short labels. In e-learning, they are often used for glossary terms or "quick tips." However, they are notoriously difficult to get right for keyboard and screen reader users.

Practice Task

1. Tab through the text and stop on a tooltip trigger.

2. Activate the trigger (Enter/Space) and identify the functional cues that announce the supplemental content.

3. While the info is visible, press Escape and verify that focus remains on the trigger while the info is dismissed.

Inaccessible Example

This "tooltip" uses CSS :hover on a non-interactive span. It cannot be seen by keyboard users and is never announced by NVDA.

Why it's broken

  • No Keyboard Access: A span is not focusable. Keyboard users can't trigger the tooltip.
  • No Programmatic Association: Even if the user could hover, the screen reader doesn't know the popup content is related to the word.
  • Dismissal Issues: A true accessible tooltip must be dismissible via the Escape key without moving focus.

Compliance Failures

  • SC 1.4.13: Content on Hover or Focus - Supplemental content cannot be dismissed, is not persistent, or is not hoverable by the user.
  • SC 2.1.1: Keyboard - Interactive information is not reachable via keyboard.

Accessible Example

The accessible version uses a <button> trigger following the Disclosure Pattern. It supports Hover, Focus, and Click activation with immediate screen reader feedback.

The POUR principles are POUR foundational to web accessibility.

You should always check your Contrast before publishing.

How it's fixed

  • Focusable Trigger: We use a <button> that is part of the tab order.
  • Aria-expanded: Tells the user if the additional information is currently visible.
  • Immediate Feedback: Activating the button via Space, Enter, or Click triggers a live region announcement of the supplemental text.
  • Aria-describedby: Ensures the content is also read automatically when the button receives focus.
  • Escape Key Support: Pressing Escape dismisses the info and returns focus to the trigger.

Compliance Success

  • SC 1.4.13: Content on Hover or Focus - The info is dismissible via Escape, stays visible while hovered, and is available to both mouse and keyboard.
  • SC 4.1.2: Name, Role, Value - Use of aria-expanded and a semantic button provides clear role and state.
