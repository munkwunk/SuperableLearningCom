Accordion

Accordions allow users to toggle the visibility of sections of content. They are vital for managing cognitive load but must communicate their expanded/collapsed state to NVDA users.

Practice Task

1. Navigate to the accessible accordion using the Tab key.

2. Listen for the "collapsed" or "expanded" state.

3. Toggle the accordion and Identify the functional cues (Role, State, Name) announced by the screen reader.

Inaccessible Example

This accordion uses generic div tags. It lacks keyboard support and provides no indication to NVDA that it can be expanded.

Perceivable
Information must be presentable to users in ways they can perceive.
Operable
User interface components and navigation must be operable.
Understandable
Operation of the UI must be understandable.
Robust
Content must be robust enough to be interpreted by a wide variety of user agents.

Why it's broken

  • No Keyboard Support: A div cannot be focused. Users who don't use a mouse are locked out.
  • No State Communication: There is no aria-expanded attribute. NVDA doesn't know if the panel is open or closed.
  • Missing Relationship: The header and panel aren't linked. NVDA cannot help the user "jump" to the content.

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The interactive header lacks a button role and expanded/collapsed state.
  • SC 1.3.1: Info and Relationships - The relationship between the header and the panel is not programmatically defined.

Accessible Example

The accessible version uses a button inside a heading, with aria-expanded and aria-controls.

Information and user interface components must be presentable to users in ways they can perceive.

User interface components and navigation must be operable.

Information and the operation of the user interface must be understandable.

Content must be robust enough that it can be interpreted by a wide variety of user agents, including assistive technologies.

How it's fixed

  • Semantic Header: We wrap the trigger in an <h3> to provide structure and landmarks.
  • State Feedback: The aria-expanded attribute updates in real-time, which NVDA announces immediately.
  • Standard Interaction: Using a real button means all keyboard users (Tab, Enter, Space) are supported by default.

Compliance Success

  • SC 4.1.2: Name, Role, Value - aria-expanded and a native <button> provide clear role and state.
  • SC 1.3.1: Info and Relationships - aria-controls and heading structure programmatically define the relationship.
