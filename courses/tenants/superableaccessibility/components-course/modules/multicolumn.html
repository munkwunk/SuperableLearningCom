Multi-Column Layouts

Complex multi-column layouts are visually engaging but can cause "reading order" issues for screen reader users if the DOM structure doesn't match the visual presentation.

Practice Task

1. Use the screen reader's linear navigation (e.g., Down Arrow) to read through the columns.

2. Identify the functional cues provided by the headings and verify that the reading order follows a logical sequence.

3. Resize your browser window to trigger a layout change and verify that the functional reading order remains consistent.

Reading order: Goal -> Problem -> Solution.

Why it's broken

  • Source Order Disconnect: Screen readers follow the DOM order, not the CSS visual order. If you use float, absolute, or order to move things visually, the reading experience becomes fragmented.
  • No Grouping: There are no headings or landmarks to tell the user that these items belong together in a logical sequence.

Compliance Failures

  • SC 1.3.2: Meaningful Sequence - The DOM order does not match the visual presentation, resulting in a nonsensical reading order for screen reader users.

How it's fixed

  • Linear DOM: The code order (Problem -> Solution -> Goal) matches the visual left-to-right columns.
  • Explicit Headings: Each column starts with an <h4>, providing a clear landmark for screen reader users to jump between sections.
  • Responsive Grid: Using CSS Grid/Flexbox allows the layout to stack on mobile without breaking the reading order.

Compliance Success

  • SC 1.3.2: Meaningful Sequence - The DOM order is aligned with the visual layout, ensuring a logical reading sequence for all users.
