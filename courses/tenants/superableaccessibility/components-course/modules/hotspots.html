Hotspot Image Interaction

Hotspots are interactive areas on an image. They are frequently used for diagrams, maps, and technical illustrations. However, because they rely on spatial positioning, they are often a nightmare for accessibility if not handled correctly.

Practice Task

1. Tab through the hotspots on the logo diagram and identify their labels and roles.

2. Activate a hotspot and identify the functional cues that indicate focus has moved to the revealed content.

3. Use the Close Details button to dismiss the information and verify that your focus returns to the marker.

Inaccessible Example

This "diagram" uses generic div elements that are absolutely positioned over the Growth for ALL logo. They are invisible to screen readers and unreachable by keyboard users.

Why it's broken

  • No Keyboard Access: These div hotspots cannot be reached via the Tab key.
  • No Alt Text or Labels: There is no descriptive text for the hotspots. An NVDA user wouldn't even know they are there.
  • Unmanaged Popups: Using alert() is a crude way to show info, but even custom "popups" often fail to move focus, leaving the user lost.

Compliance Failures

  • SC 1.1.1: Non-text Content - The image and interactive areas lack descriptive text alternatives.
  • SC 2.1.1: Keyboard - Hotspots implemented as div elements are not focusable or operable via keyboard.
  • SC 2.4.3: Focus Order - There is no focus management for revealed content, and alerts cannot be dismissed via keyboard in a controlled way.
  • SC 4.1.2: Name, Role, Value - The interactive areas lack a programmatic role and accessible name.

Accessible Example

The accessible version uses a Hotspot Container with semantic <button> markers. Activating a marker moves focus directly into the revealed content.

How it's fixed

  • Semantic Triggers: Each hotspot is a <button> with a descriptive aria-label.
  • Focus Management: When a hotspot is activated, focus is moved into the newly visible info panel. This ensures NVDA reads the description immediately.
  • Contextual Return: Closing the info panel returns focus to the exact hotspot marker that triggered it, maintaining the user's place on the page.

Compliance Success

  • SC 1.1.1: Non-text Content - Meaningful image alternative and descriptive labels for interactive hotspots.
  • SC 2.1.1: Keyboard - Each hotspot is a native <button>, ensuring keyboard focus and activation.
  • SC 2.4.3: Focus Order - Focus moves logically to revealed content and returns to the trigger on close.
  • SC 4.1.2: Name, Role, Value - Use of native buttons and aria-label provides correct role and name.
