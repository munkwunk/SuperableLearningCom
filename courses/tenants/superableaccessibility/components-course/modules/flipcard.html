Flip Cards

Flip cards are a common e-learning pattern used to reveal hidden information. However, they are often implemented using only visual transitions, making them completely inaccessible to screen reader and keyboard users.

Practice Task

1. Use the keyboard to navigate to the accessible flip card.

2. Activate the card and identify the functional cues that indicate the state change (front vs. back).

3. Verify that the focus management provides immediate feedback after the card flips.

Inaccessible Example

These flip cards use div tags with click listeners. They cannot be focused by keyboard and provide no state information to NVDA.

Why it's broken

  • No Keyboard Access: The div elements are not focusable. Keyboard users cannot "flip" them.
  • No Semantic Role: NVDA sees these as plain text, not interactive components.
  • No State Announcement: When the "flip" happens visually, NVDA users receive no feedback that content has changed.

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The interactive element lacks a semantic role and state.
  • SC 2.1.1: Keyboard - The component is not operable via keyboard.
  • SC 2.4.3: Focus Order - Lack of focus management prevents a logical navigation path when content changes.

Accessible Example

The accessible version uses a button to trigger the flip and manages the visibility of the front and back content using the hidden attribute.

How it's fixed

  • Semantic Button: We use a real <button> that is naturally focusable and provides a clear action label.
  • Focus Management: When the card flips, focus is moved to the revealed container. This ensures the user is immediately aware of the new content.
  • State Management: The hidden attribute ensures that only the relevant side is visible to both the eye and the screen reader.

Compliance Success

  • SC 4.1.2: Name, Role, Value - Using a <button> provides the correct role and state.
  • SC 2.1.1: Keyboard - The <button> is natively keyboard accessible.
  • SC 2.4.3: Focus Order - Moving focus to the revealed content ensures a logical path.
