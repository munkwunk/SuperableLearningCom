Click-to-Reveal Panels

Click-to-reveal is a fundamental e-learning interaction. It allows learners to explore content at their own pace. However, if implemented poorly, it can be a "black box" for screen reader users who may not know new content has appeared.

Practice Task

1. Tab to the accessible reveal panels.

2. Use the Spacebar or Enter key to toggle the content.

3. Identify the functional cues (Role, State, Name) announced by the screen reader. Then, continue tabbing to read the revealed content.

Inaccessible Example

This panel uses a generic div. It is not focusable by keyboard, and there is no indication that it contains interactive content or what its current state is.

Why it's broken

  • Non-Interactive Element: A div does not appear in the tab order. Keyboard users will skip right over this.
  • No State Feedback: NVDA users have no way of knowing if the panel is open or closed because there is no aria-expanded attribute.
  • No Role: The screen reader sees this as plain text, not a button or a disclosure widget.

Compliance Failures

  • SC 4.1.2: Name, Role, Value - The trigger lacks a semantic role and expanded/collapsed state.
  • SC 2.1.1: Keyboard - The div trigger is not focusable or operable via keyboard.
  • SC 1.3.1: Info and Relationships - The relationship between the trigger and the revealed content is not programmatically defined.

Accessible Example

The accessible version uses a <button> and follows the Disclosure pattern. This ensures that the interaction is clear, focusable, and announces its state correctly.

Information and user interface components must be presentable to users in ways they can perceive.

User interface components and navigation must be operable.

Information and the operation of the user interface must be understandable.

Content must be robust enough that it can be interpreted by a wide variety of user agents, including assistive technologies.

How it's fixed

  • Semantic Button: We use a real <button> that is automatically part of the tab order.
  • Aria-Expanded: The aria-expanded attribute tells the user if the content is currently visible or hidden.
  • Aria-Controls: This attribute creates a programmatic link between the button and the content panel it manages.

Compliance Success

  • SC 4.1.2: Name, Role, Value - Native <button> and aria-expanded provide clear role and state.
  • SC 2.1.1: Keyboard - Using a native <button> ensures the interaction is focusable and operable via keyboard.
  • SC 1.3.1: Info and Relationships - aria-controls programmatically links the trigger to its content.
