Matching Interactions

Matching tasks (like drag-and-drop) are common in e-learning. However, the method of interaction can determine whether a learner is included or excluded. Following Universal Design for Learning (UDL), we should provide multiple means of action and expression.

Practice Task

1. Complete the matching game using only your keyboard.

2. Activate "Check Answers" and Identify the functional cues (Role, State, Name) announced by the screen reader.

3. Notice how the visual score display and the NVDA announcement work together to provide clarity.

Inaccessible Drag-and-Drop

Why it's broken

  • No Keyboard/NVDA Access: The drag-and-drop items and drop zones are completely non-interactive for non-mouse users.
  • Silent Feedback: When "Submit" is pressed, the screen reader remains silent, leaving the user with no idea of their performance.
  • Violation of UDL: It provides only one way to interact (mouse) and one way to receive feedback (visual).

Compliance Failures

  • SC 2.1.1: Keyboard - The drag-and-drop interaction is only available via mouse.
  • SC 4.1.3: Status Messages - The results of the matching task are not announced to screen readers.

Robust Accessible Alternative: Matching Game

This version uses the same POUR principles but employs native <select> menus. It provides multi-modal feedback (visual and auditory) for a truly universal experience.

How it's fixed (UDL Focus)

  • Multiple Means of Action: Accessible via mouse, keyboard, and screen reader using standard patterns.
  • Immediate Multi-Modal Feedback: Correct/Incorrect states are shown with colors (visual), icons (visual), and text (auditory via live regions).
  • Persistent Results: The final score is displayed on-screen for everyone and announced specifically for NVDA users.

Compliance Success

  • SC 2.1.1: Keyboard - The matching task is operable using standard form controls (select menus).
  • SC 4.1.3: Status Messages - The score is announced via an ARIA live region upon submission.
